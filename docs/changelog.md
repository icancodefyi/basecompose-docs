## How We Version

Basecompose follows [Semantic Versioning](https://semver.org/):

- **MAJOR** (e.g., 2.0.0) - Breaking changes
- **MINOR** (e.g., 1.1.0) - New features (backward compatible)
- **PATCH** (e.g., 1.0.1) - Bug fixes

**Example:**
- `1.0.0` - Initial release
- `1.1.0` - Added PostgreSQL support (new feature)
- `1.1.1` - Fixed MongoDB connection bug (patch)
- `2.0.0` - Major API changes (breaking change)

---

## Release Schedule

- **Patch releases:** As needed (bug fixes)
- **Minor releases:** Monthly (new features)
- **Major releases:** Annually or as needed (significant changes)
